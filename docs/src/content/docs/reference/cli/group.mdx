Groups allow you to manage permissions for multiple users at once.

## Commands

### Group Management

| Command | Description |
|---------|-------------|
| `create` | Create a new group |
| `list` | List groups in a workspace |
| `delete` | Delete a group |
| `update` | Update group name or description |

### Membership

| Command | Description |
|---------|-------------|
| `add-member` | Add a user to a group |
| `remove-member` | Remove a user from a group |
| `list-members` | List members of a group |

### Group Permissions

| Command | Description |
|---------|-------------|
| `set-permission` | Set workspace permission for a group |
| `remove-permission` | Remove workspace permission |
| `get-permission` | Get workspace permission |
| `list-permissions` | List all group permissions on workspace |
| `set-project-permission` | Set project permission for a group |
| `remove-project-permission` | Remove project permission |
| `get-project-permission` | Get project permission |
| `list-project-permissions` | List group permissions on project |
| `set-env-permission` | Set environment permission for a group |
| `remove-env-permission` | Remove environment permission |
| `get-env-permission` | Get environment permission |
| `list-env-permissions` | List group permissions on environment |

## Group Management

### group create

Create a new group.

```bash
zopp group create [-w <WORKSPACE>] <NAME> [-d <DESCRIPTION>]
```

**Example:**

```bash
zopp group create -w acme-corp backend-team -d "Backend engineering team"

# Output:
# ✓ Group 'backend-team' created
```

### group list

List all groups in a workspace.

```bash
zopp group list [-w <WORKSPACE>]

# Output:
# Groups:
#   backend-team    Backend engineering team
#   frontend-team   Frontend developers
#   devops          DevOps and infrastructure
```

### group update

Update a group's name or description.

```bash
zopp group update [-w <WORKSPACE>] <NAME> [--new-name <NAME>] [-d <DESCRIPTION>]
```

**Example:**

```bash
zopp group update -w acme-corp backend-team --new-name api-team
```

### group delete

Delete a group.

```bash
zopp group delete [-w <WORKSPACE>] <NAME>

# Output:
# ✓ Group 'old-team' deleted
```

## Membership

### group add-member

Add a user to a group by email.

```bash
zopp group add-member [-w <WORKSPACE>] -g <GROUP> <EMAIL>
```

**Example:**

```bash
zopp group add-member -w acme-corp -g backend-team alice@company.com

# Output:
# ✓ Added alice@company.com to 'backend-team'
```

### group remove-member

Remove a user from a group.

```bash
zopp group remove-member [-w <WORKSPACE>] -g <GROUP> <EMAIL>
```

### group list-members

List all members of a group.

```bash
zopp group list-members [-w <WORKSPACE>] -g <GROUP>

# Output:
# Members of 'backend-team':
#   alice@company.com
#   bob@company.com
#   charlie@company.com
```

## Common Patterns

### Set up team access

```bash
# Create teams
zopp group create -w acme-corp backend-team -d "Backend engineers"
zopp group create -w acme-corp frontend-team -d "Frontend engineers"
zopp group create -w acme-corp devops -d "DevOps team"

# Add members
zopp group add-member -w acme-corp -g backend-team alice@company.com
zopp group add-member -w acme-corp -g backend-team bob@company.com

# Set permissions
zopp group set-project-permission -w acme-corp -p api-backend -g backend-team --role write
zopp group set-project-permission -w acme-corp -p web-frontend -g frontend-team --role write
zopp group set-permission -w acme-corp -g devops --role admin
```

### Restrict production access

```bash
# Backend team has write everywhere
zopp group set-permission -w acme-corp -g backend-team --role write

# But only read on production
zopp group set-env-permission -w acme-corp -p api-backend -e production -g backend-team --role read
```

## Notes

- Group permissions apply to all members' principals
- A user can be in multiple groups
- Group permissions are combined with individual permissions
- When a user is removed from a group, they lose that group's permissions
