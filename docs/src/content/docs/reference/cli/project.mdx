Projects group related environments, typically representing an application or service.

## Commands

| Command | Description |
|---------|-------------|
| `list` | List projects in a workspace |
| `create` | Create a new project |
| `get` | Get project details |
| `delete` | Delete a project |

---

## project list

List all projects in a workspace.

### Usage

```bash
zopp project list -w <WORKSPACE>
```

### Options

| Option | Description |
|--------|-------------|
| `-w, --workspace <NAME>` | Workspace name |

### Example

```bash
zopp project list -w acme-corp

# Output:
# Projects:
#   api-backend
#   web-frontend
#   mobile-app
```

## project create

Create a new project in a workspace.

### Usage


| Argument | Description |
|----------|-------------|

### Options

| Option | Description |
|--------|-------------|
| `-w, --workspace <NAME>` | Workspace name |

## project delete

Delete a project and all its environments and secrets.

### Usage

```bash
zopp project delete <NAME> -w <WORKSPACE>
```

### Arguments

| Argument | Description |
|----------|-------------|
| `NAME` | Project name |

### Options

| Option | Description |
|--------|-------------|
| `-w, --workspace <NAME>` | Workspace name |

### Example

```bash
zopp project delete api-backend -w acme-corp

# Output:
# ✓ Project 'api-backend' deleted
```

:::caution
Deleting a project removes all environments and secrets within it. This action cannot be undone.
:::

## Notes

- Project names must be unique within a workspace
- Projects inherit permissions from the workspace unless overridden
- A project must be empty (no environments) or you must confirm deletion
